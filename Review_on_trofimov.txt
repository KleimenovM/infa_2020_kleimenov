Ревью на работу Ивана Трофимова

Всесь код разделён на две части:
первая часть состоит из функций,
вторая - сплошной код, где используются функции из первого блока

Есть несколько недостатков:
- использование несколько раз вложенных списков
- широкое использование глобальных переменных, которые определяются не в шапке программы, а во второй части со сплошным
 кодом (выходит, что глобальные переменные определяются после того, как появляются в коде
- отсутствие комментариев о требуемых функциями переменных
- нигде не описаны правила игры
- автор не поместил в GitHub требуемые программой файлы
- существует некоторое количество потенциально недостижимых мест, которые обходятся большим количеством условий в программном коде
- в коде есть одноимённые глобальные и локальные переменные, что делает некоторые места программы неоднозначными

С другой стороны, работа не лишена большого количества достоинств:
- оригинально реализована динамика игры
- существуют различные элементы геймплея (в т.ч. может появляться босс игры)
- несмотря на кажущуюся на первый взгляд хаотичность кода, все неоднозначные элементы прокомментированы
- программа выглядит в целом структурированной и код читабелен
- при запуске игры на экране показывается время, очки и количество шаров
